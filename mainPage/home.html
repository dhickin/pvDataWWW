This is the group home page of the EPICS v4 Working Group.

EPICS v4 is a set of computer communications protocols, and a software framework, for high performance distributed control, message passing, and high level software services, as may be used in large scientific instruments and industrial plants. EPICS v4 was previously called "PvData", hence the name of the sourceforge project.

The EPICS v4 working group is a collaborative effort of members invited by Brookhaven Lab, to bring EPICS v4 to its full potential. BNL and PSI support is subject to the EPICS v4 Charter.

Status

Status 4-Aug-2011

The implementation statuses of the core packages of Epics v4, pvData, pvAccess, pvIOC, and PvService, given by language support, are:

Java

pvData, pvAccess, pvIOC, and PvService packages exist and are being developed as Eclipse projects, made available through sourceforge project epics-pvdata. The Java version of the pvIOC is named "javaIOC".

The following services are implemented. Presently they can be found in PvService:

  • itemFinder - Get a list of channel names and properties. Both client and server are implemented
  • gather - get/put/monitor a set of V3 channels, Both client and server are implemented
  • model - Guobao can give status

C++

pvDataCPP and pvAccessCPP are completed and available via sourceforge mercurial projects.

pvIOCCPP is started. It is implemented as EPICS V3 IOC applications. It has a pvAccess server that accesses EPICS V3 records. Much work remains.

pvServiceCPP has client side support for gather. It requires that a Java gather server be running.

Python

pvData and pvAccess - Guobao is working on wrapping pvDataCPP and pvAccessCPP

pvService - The client side of gather is implemented by using the C++ gather client support. Note about Python gather client:

A performance tests shows results equivalent to C++. This is because all the real work is done by C++ code not by Python. The V3 channel data is provided as a numPy 1-d double array that uses the underlying double array from the C++ array. It is expected that the Python client will pass the numPy array to other numPy or sciPy based Python code.
